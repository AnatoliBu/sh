# References

`references/` is the source-of-truth layer.

Keep it mostly flat. It should answer:

- which sources are authoritative;
- which sources are secondary;
- how to resolve conflicts;
- which materials are unsafe or too weak to promote into skills/rules.

## Catalog

- [[source-quality|Source quality]]
- [[sysadmin|Sysadmin references]]
- [[analytics|Analytics references]]

## Rule

A reference document should not become an agent behavior spec. When a source produces an actionable rule, move the behavior into the relevant domain folder, for example `sysadmin/rules` or `analytics/skills`.
